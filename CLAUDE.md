# ClassicMCP — инструкции для Claude Code

Этот проект автоматизирует лабораторные работы по ТАУ в программе CLASSiC 3.2
через MCP-сервер `classic-automation`.

## Файлы и папки

- `tasks/` — файлы заданий (LP_vXX.doc / LP_vXX.docx)
- `reports/` — готовые отчёты (LP_vXX_done.docx) и скриншоты (vXX_shots/)
- `classic/` — CLASSiC 3.2 и .mdl модели

## Типичное задание

Если пользователь говорит «обработай вариант 17» или «сделай лабу»:
1. Вызови `parse_task_file("tasks/LP_v17.docx")` — получишь параметры И `q9_text`
2. Вызови `calculate_transfer_functions(...)` с полученными параметрами
3. Прочитай `q9_text`, рассуди какие из задач 1–8 содержат запрошенный концепт,
   сформируй `q9_answer` (например `"4, 5, 6, 10, 11"`)
4. Вызови `write_mdl_file(...)` → `run_classic_gui(...)` → `fill_report_docx(..., q9_answer=...)`

Или одной командой: `run_full_pipeline("tasks/LP_v17.docx", q9_answer="4, 5, 6, 10, 11")`

## Что автоматизировано

Отчёт заполняется полностью автоматически:
- Формулы WP(s), Ф(s), Фe(s), eуст, Ккр
- Таблицы коэффициентов ПФ и блоков CLASSiC
- Скриншоты (схема, переходные процессы, ЛЧХ и др.)
- Жирным выделяются правильные ответы на тесты: Q1, Q3, Q10, Q12, Q13, Q14, Q15
- Q9 заполняется через `q9_answer` (Claude определяет из `q9_text`)

Вручную остаётся только: поля «Вывод о совпадении» после каждой задачи.
