# ClassicMCP

Автоматизация лабораторных работ по ТАУ (Теория автоматического управления) в программе **CLASSiC 3.2** через MCP-сервер для Claude Code.

---

## Быстрый старт

### 1. Установка зависимостей

```bash
cd classic_automation
pip install -r requirements.txt
```

### 2. Регистрация MCP-сервера в Claude Code

```bash
claude mcp add classic-automation python classic_automation/server.py
```

Или добавь вручную в `.claude/settings.json` проекта:

```json
{
  "mcpServers": {
    "classic-automation": {
      "command": "python",
      "args": ["classic_automation/server.py"]
    }
  }
}
```

### 3. Проверка

Запусти в Claude Code:

> "Проверь зависимости classic-automation"

Claude вызовет инструмент `check_dependencies` и покажет статус каждого компонента.

### 4. Подготовка шаблона

В папке `classic/` должен лежать файл `template.mdl` — модель варианта 17, созданная вручную в CLASSiC 3.2 с параметрами:

| Блок | Передаточная функция | Параметры |
|------|----------------------|-----------|
| W2   | K1                   | K1 = 100  |
| W3   | K3 / (T3·s + 1)     | K3 = 2.5, T3 = 0.5 |
| W4   | K4 / (T4·s + 1)     | K4 = 0.2, T4 = 0.05 |
| W5   | K5 / s              | K5 = 0.05 |

## Использование

Типичный диалог с Claude Code:

> "Обработай задание tasks/LP_v17.doc"

Claude сам выберет нужные инструменты в правильном порядке. Можно также вызывать инструменты по отдельности:

> "Рассчитай передаточные функции для варианта 17"  
> "Сгенерируй MDL для варианта 3"  
> "Запусти CLASSiC и сними скриншоты для var17.mdl"

## Структура проекта

```
ClassicMCP/
├── classic/                  — CLASSiC 3.2 (Win16 через otvdm)
│   ├── CLASSiC32.exe
│   ├── CLASSiC.ini
│   ├── template.mdl          — шаблон модели (создаётся вручную)
│   └── vdm/                  — otvdm (эмулятор Win16)
├── classic_automation/       — MCP-сервер и все модули
│   ├── server.py             — FastMCP-сервер (точка входа)
│   ├── config.py             — пути (вычисляются автоматически)
│   ├── parser.py             — парсинг .doc, таблица вариантов
│   ├── calculator.py         — расчёты через SymPy
│   ├── mdl_writer.py         — генерация .mdl файла
│   ├── mdl_patcher.py        — бинарный патч template.mdl
│   ├── classic_gui.py        — автоматизация CLASSiC через pyautogui
│   ├── report.py             — заполнение .docx отчёта
│   └── main.py               — CLI-запуск без MCP
├── tasks/                    — .doc файлы заданий
├── models/                   — сгенерированные .mdl файлы
└── reports/                  — готовые отчёты и скриншоты
```

## Доступные инструменты MCP

| Инструмент | Что делает |
|------------|------------|
| `check_dependencies` | Проверяет Python-пакеты, CLASSiC.exe, рабочие папки |
| `list_task_files` | Перечисляет .doc/.docx файлы в папке заданий |
| `parse_task_file` | Читает .doc, извлекает номер варианта и параметры |
| `get_variant_params` | Возвращает параметры варианта из таблицы (без файла) |
| `calculate_transfer_functions` | Считает WP(s), Ф(s), Фe(s), Гурвиц, K1кр, eуст |
| `write_mdl_file` | Генерирует .mdl модель для CLASSiC |
| `run_classic_gui` | Запускает CLASSiC, снимает 8 скриншотов |
| `fill_report_docx` | Вставляет расчёты и скриншоты в .docx шаблон |
| `run_full_pipeline` | Полный пайплайн одной командой (парсинг → отчёт) |

### Порядок вызовов

```
get_variant_params / parse_task_file
        ↓
calculate_transfer_functions
        ↓
write_mdl_file
        ↓
run_classic_gui
        ↓
fill_report_docx
```

## Что уже работает

- **Расчёты** (`calculate_transfer_functions`) — SymPy считает WP(s), Ф(s), Фe(s), критерий Гурвица, K1кр, eуст.
- **Генерация MDL** (`write_mdl_file`) — бинарный патч `template.mdl` для всех 20 вариантов.
- **Запуск CLASSiC, 9 скриншотов** (`run_classic_gui`) — схема, текстовая форма, характеристики (4 квадранта), рамповый отклик (Линейное 0.1), скриншот при K1кр.
- **Кадрирование панелей** — границы 4 квадрантов и разделители (MX/MY) определены пиксельным сканированием; каждая панель захватывается точно по своим границам.
- **Парсинг** (`parse_task_file`) — `.docx` и файлы с `v17`/`вариант 17` в имени или тексте. Для `.doc` требует LibreOffice; без него — автоматический fallback на имя файла (без краша).
- **Заполнение отчёта** (`fill_report_docx`) — конвертирует `.doc` → `.docx` через Word COM и заполняет:
  - формулы WP(s), Ф(s), Фe(s), eуст (ступенчатое и рамповое), Kкр
  - таблицы коэффициентов ПФ для WP, Ф, Фe (авто-вставка строк для систем выше 2-го порядка)
  - имя MDL-файла в строках «Модель сохранена в файле» и «Модель: "….MDL"»
  - текстовую таблицу блоков CLASSiC (K1, K3/T3, K4/T4, K5)
  - частотные показатели качества (задача 13) через numpy
  - правильные ответы на тесты выделяются жирным: Q1 (класс + астатизм), Q3, Q10, Q12, Q13, Q14, Q15
  - Q9 заполняется через параметр `q9_answer` — Claude читает `q9_text` из `parse_task_file` и рассуждает
  - XML run-splitter (`_bold_text_in_para`) болдит любую подстроку параграфа, даже если Word порезал её на части
  - скриншоты вставляются перед Рис.1а, Рис.3, Рис.4, Рис.5, Рис.6
- **Полный пайплайн** (`run_full_pipeline` / `main.py`) — проверен на варианте 17 от `.docx` до готового `.docx` отчёта.

## Известные проблемы

- **Детектор отрисовки** (`_wait_for_render`): периодически выдаёт «таймаут ожидания изменения» — CLASSiC отрисовывается быстрее, чем детектор успевает засечь изменение. Скриншот всё равно снимается корректно.
- **Поля «Вывод о совпадении»** — везде вручную (содержательный текст).

## Известные ограничения

- **Только этот ПК / сервер с CLASSiC**: `run_classic_gui` требует установленного CLASSiC 3.2 и `template.mdl`. На машине без GUI не работает.
- **Мониторы**: CLASSiC автоматически перемещается на основной монитор (`200, 150`). При другом расположении поправь `_move_to_primary(x=..., y=...)` в `classic_gui.py`.
- **Количество вариантов**: `VARIANT_TABLE` в `parser.py` содержит 20 вариантов с примерными параметрами — уточни свою таблицу.
- **Экранное разрешение и размер окна**: пропорции кадрирования (`_quad_crops`) и координата блока K1 (`_critical_shot`) откалиброваны по окну 1375×994 px на 3440×1440 @ 125% DPI. При другом разрешении или размере окна CLASSiC потребуется повторное сканирование пропорций.
